What’s in the box

Each piece is text and scripts you can merge into a real repo or your user ~/.cursor config. Secrets stay on your machine.

Rules

Project guardrails

Atlassian, browser (Charlotte/Playwright), token efficiency, data-eng workflows, and more — as .mdc files for Cursor rules.

Skills

Clarify, plan, present

Brainstorm, planning-brief, and presentation-creation (markdown → static HTML deck) with clear triggers in each skills.md.

Subagents

Single-role prompts

Implementer, tester, reviewer, and ship-writer: invoke one at a time or in sequence for a repeatable pipeline.

MCP

Example servers

mcp.json.example + MCP.md describe code-review-graph, Charlotte, Playwright, and optional Tableau with placeholders only.

Hooks

Telemetry & guard

Optional local JSONL logging and a Langfuse-oriented bundle with MCP write guard. Fail-open by design.

A default path through the work

Not a product requirement — a convention the README encodes. Skip steps when the task is small.

user prompt → brainstorm → planning_brief
↓
implementer → tester → reviewer → ship_writer

Decks and docs can also branch from the initial prompt (e.g. presentation-creation).

Get the kit

Clone, then follow the install guide (SETUP.md) in the repository.

git clone https://github.com/abhinavsinhanutan/Cursor-Workbench.git
cd Cursor-Workbench
export CURSOR_WB_ROOT="$(pwd)"
